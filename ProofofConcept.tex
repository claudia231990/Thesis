In diesem abschnitt wird der bestehende Ist-Zustand an einem Produkt wie CSS oder  HTM näher beschrieben (welche FOSS wird verwendet, welche Tools im Devops-Bereich). Die beschriebenen fachlichen Rahmenbedinungen werden nun an dieser Stelle anhand dieses IST-Zustand angewandt, um mehrere geeignete Lösungsstrategien für Veränderungen innerhalb von FOSS darzustellen. Dabei werden insbesondere Prozessmodelle konzepiert um die vorhandenen Problematiken innerhalb des IST-Zustandes anhand der neu gewonnenen Erkenntnisse als ein Vorgehen vorzustellen.
